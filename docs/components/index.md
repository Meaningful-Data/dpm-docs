# 5 Metamodel components

As described in chapter 3 of this document, DPM metamodel consists of four main components.
Sections of this chapter explain each of these components in detail, one by one. Links embedded in
text shall help navigation as components are heavily interrelated.

The four components are:

- [5.1 Glossary](glossary.md) — definition and management of terms and notions used to describe
  information requirements.
- [5.2 Grouping, rendering, and packaging of information requirements](rendering-packaging.md) —
  tabular representation of information requirements and their packaging in Frameworks and Modules.
- [5.3 Identification and description of each reported value](variables.md) — Variables that
  uniquely identify and describe each reportable value.
- [5.4 Operations on data](operations.md) — data quality checks (validations) and data
  transformation / derivation rules.
